## 1. 重做卡片飞入轨迹与节奏 — `src/pages/Index.tsx`
当前问题：
- 现在卡片只是简单 `translateY(-60vh)` 到终点，起点不准，终点偏下。
- 动效只靠 `transition: all`，没有“从视频中的扑克牌抽离出来”的空间感，所以显得生硬卡顿。

改法：
- 把卡片飞入拆成两段状态，而不是现在的一段硬切：
  - 初始态：卡片位于页面上方偏中间，也就是视频里扑克牌的大致位置；
  - 飞入态：带 `translate3d + scale + rotateX/rotateZ + blur/opacity` 的组合过渡，落到页面中间偏上的扇形终点。
- 参考你提到的 `Aideo Studio` 里“弹窗飞入输入框”的做法，用“先算终点、再从起点飞向终点”的思路，改成卡片从无到有飞入，而不是直接渲染后再平移。
- 给每张卡片做轻微错峰延迟，但总时长控制在 700-900ms 内，曲线改为更顺滑的 `cubic-bezier(0.4, 0, 0.2, 1)` / 类 spring 的组合，避免现在的顿挫。
- 卡片最终容器整体上移，确保扇形中心对齐标题下方，而不是压到页面偏下区域。
- 如果 loop 视频文件过大导致解码/切换卡顿，实施时会同时压缩替换视频2，保证预览顺滑。

## 2. 重构卡片底部蒙层结构 — `src/components/TemplateCard.tsx`
当前问题：
- 底部蒙层缺少明确的“清晰玻璃底色 + 上边缘渐变模糊过渡”两层结构。
- hover 后的蒙层和非 hover 态不是同一套层，导致 button 出现时卡顿、边缘太低、样式不一致。
- Try this 按钮圆角也不对。

改法：
- 把底部区域改成统一的双层结构：
  - 上层：单独的渐变模糊过渡带，只负责上边缘柔化；
  - 下层：固定清晰可见的 frosted glass 实体蒙层，非 hover 和 hover 共用这一层。
- 文字态和按钮态只切换内容，不再切换整个蒙层容器，避免 hover 时重排和闪断。
- 提高渐变模糊带的位置，让上边缘过渡更高、更自然，和你之前认可的方向一致。
- Try this 按钮圆角改为 `16px`。
- 按钮出现方式改成内容层的 opacity/translate 微过渡，而不是整块容器突然变化，解决“按钮出现卡顿”。

## 3. 主标题/副标题全程常驻 — `src/components/HeroSection.tsx` + `src/pages/Index.tsx`
当前问题：
- `HeroSection` 在 `ready` 阶段被隐藏了，和你的要求相反。
- 现在标题在不同阶段的位置逻辑也不够稳定。

改法：
- 去掉“ready 阶段标题消失”的逻辑，主副标题在 `intro / loop / cards-fly / ready` 全程显示。
- 只做尺寸和位置微调，不做 opacity 消失：
  - 视频阶段：维持大标题、居中；
  - 卡片飞入后：标题保持可见，但整体位置根据输入框是否出现做轻微上移，保证与卡片间距恒定。
- 保留并加强标题阴影，确保在两个视频背景上始终清晰。

## 4. 让“从视频中飞入网页”的沉浸感成立 — `src/pages/Index.tsx` + 视频资源
当前问题：
- 现在更像“网页里生成卡片”，不是“视频里的扑克牌变成可用模板卡片”。
- 视频2首轮结束后触发点有了，但视觉承接不够。

改法：
- 把卡片的飞入起点锁定在视频最后一帧扑克牌所在的区域，按 1454×900 当前视口重新校准，不再从过低位置起飞。
- 在卡片初始态加入更强的“从场景里抽出来”的视觉线索：
  - 更小初始 scale
  - 更强透视/倾斜
  - 初始轻微模糊 + 低透明度
  - 落位后恢复清晰
- 卡片飞入时保留标题不消失，形成“视频中的牌 → 网页中的模板卡片”这一连续叙事。
- 如有必要，实施时会顺带微调 loop 视频切换方式，避免 `onEnded` 后重复播放造成的硬切感。

## 技术说明
- 这次核心不是再调几个 `margin-top`，而是把“动画状态机 + 卡片蒙层结构”重做正确。
- 飞入动画会避免继续使用单一 `transition: all`，而是明确控制 `transform / opacity / filter`，这样更顺滑，也更接近你指定参考项目的 handoff 感。
- 蒙层必须用“固定底层 + 可变内容层”结构，才能保证 hover 前后视觉一致、不跳、不糊错地方。
