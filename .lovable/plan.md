## 1. 解决输入框遮挡标题：让标题和卡片整体下移，但位置固定 — `src/pages/Index.tsx`
当前代码问题：
- 输入框是 `absolute top-0 + paddingTop: 64px`。
- 标题卡片舞台现在固定在 `top: 38%`，导致输入框出现后直接压住标题。
- 这说明“舞台层”和“输入框层”虽然分离了，但舞台锚点太高。

改法：
- 保留输入框绝对定位。
- 把非 intro 阶段的标题+卡片舞台整体下移，改成更稳定的固定锚点，而不是继续停在现在这个会被遮挡的位置。
- 标题和卡片作为一个整体下移，不单独推标题，避免标题和卡片间距被破坏。
- 保持：
  - 输入框到标题：64px
  - 标题到卡片：64px

结果：
- 输入框出现后不再遮挡主副标题。
- 标题不会乱跳，只会在 video2 出现后进入一个固定位置。

## 2. 重做卡片飞入：不再在终点容器里“原地出现” — `src/pages/Index.tsx`
当前代码问题：
- 现在虽然写了外层 `translateY(-45vh)`，但卡片仍然是在最终舞台内部被渲染出来的。
- 也就是说，用户看到的仍是“中心区域先生成卡片，再做一点变形”，所以视觉上就是突然出现、还会卡。
- 真正缺失的是：起点和终点没有用页面真实坐标分离。

改法：
- 把“飞行中的卡片层”和“落地后的卡片层”拆成两层：
  1. 飞行层：绝对定位在页面里，起点直接放在输入框出现的位置 / 视频扑克牌区域；
  2. 落地层：单独放在最终中心扇形位置，只在飞行完成后显示。
- 动画不再依赖“卡片已经在终点容器里，再 translate 回去”，而是：
  - 起点：页面顶部输入框区域，极小、透明、带 blur、带 rotateY/rotateX/rotateZ
  - 终点：画面中心偏上的扇形位置，scale 1、blur 0、opacity 1、翻转归正
- 外层飞行容器整体从顶部飞到中心；
- 每张卡片同时做：
  - scale 放大
  - rotateY 翻转回正
  - 少量 rotateZ 扇形展开
  - opacity / blur 丝滑过渡
- 动画结束后再切换到稳定的最终卡片层，避免 hover 和飞行动画互相打架。

结果：
- 卡片会真正从“输入框出现的位置 / 视频上方扑克牌区域”飞到画面中心。
- 不再是中心突然冒出来。
- 从无到有、翻转放大的过程会明显成立。

## 3. 专门处理卡顿：减少首帧重排和同层混合渲染 — `src/pages/Index.tsx`
当前代码问题：
- 现在卡片一出现就在最终 flex 布局里做 3D 变换，浏览器会同时处理布局、绘制、滤镜、hover 样式，容易卡。
- 这是你一直看到“突然出现 + 掉帧”的原因之一。

改法：
- 飞行阶段使用独立绝对层，不参与最终卡片布局。
- 飞行卡片统一加上：
  - `will-change: transform, opacity, filter`
  - `backface-visibility: hidden`
  - `transform-style: preserve-3d`
  - `translate3d(...)`
- 错峰延迟缩短，避免拖泥带水。
- 最终静态卡片层只负责展示，不承担飞行动画。

结果：
- 首帧不会再像现在这样“先蹦出来一下”。
- 动画会更连续，更接近你要求的从视频上方抽离出来的感觉。

## 4. 标题字号固定，不允许输入框出现后变小 — `src/components/HeroSection.tsx`
当前代码要求确认：
- 你已经明确说了：主副标题文字不应该在输入框出现后变小。

改法：
- `HeroSection` 不再根据 `phase` 或 `showPanel` 改字号。
- 主标题、副标题字号固定，只允许位置变化，不允许缩放变化。

结果：
- 标题视觉保持一致，不会再出现输入框一出来标题缩水的问题。
