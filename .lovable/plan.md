## 实施方案

### 1. 彻底重做卡片飞入结构 — `src/pages/Index.tsx`

不再保留现在这个“居中 flex 飞行层”。

改成两层：

- **飞行层**：`position: fixed`，独立覆盖全屏，不参与最终卡片布局
- **落地层**：只负责最终静态扇形卡片和 hover

飞行层里 4 张卡片分别独立绝对定位：

- 同一高度起点（输入框上方位置）：`12vh`
- 4 个不同 X 起点：按视频中扑克牌大致分布设置
- 终点：当前卡片扇形的 4 个真实落点

每张卡片动画统一做：

- `scale(0.08) -> 1`
- `rotateY(180deg) -> 0deg`
- `rotateX(10~14deg) -> 0deg`
- `blur(10px) -> 0`
- `opacity(0 -> 1)`

并且：

- 外层飞行容器不再居中生成
- 每张卡片自己从顶部真实坐标飞到各自扇形终点
- 动画结束后再切到静态落地层，hover 不再干扰飞行动画

### 2. 动效节奏对齐 `Aideo Studio` 弹窗飞入质感 — `src/pages/Index.tsx` + `src/index.css`

这次不再用现在这套 1.4s keyframe + 模糊中心突现的方式。

改为更接近参考项目的节奏：

- 主位移/翻转缓动：`cubic-bezier(0.4, 0, 0.2, 1)`
- 单张卡片主时长按参考项目的丝滑节奏统一
- 4 张卡片保留很轻的错峰（例如 40~60ms），但不会拖沓

关键点：

- 用真实坐标动画，不再用终点容器内相对偏移假装飞入
- 用 `animationend` / 明确结束信号切换落地层，不再靠粗暴 `setTimeout` 硬切
- 飞行卡统一加：
  - `will-change: transform, opacity, filter`
  - `transform-style: preserve-3d`
  - `backface-visibility: hidden`

### 3. 修复“图片加载卡顿”这个真正根因 — `src/pages/Index.tsx` + `src/components/TemplateCard.tsx`

这次会把卡片封面图预加载好再开飞行动画。

做法：

- 在首页进入 loop 阶段时，提前预加载 4 张模板图
- 用 `Image()` + `decode()` 等待图片解码完成
- 图片未准备好前，不启动卡片飞入
- `TemplateCard` 首屏 4 张卡改为高优先级加载，不再使用 `loading="lazy"`

结果：

- 飞行时不会因为图片边飞边解码而掉帧
- 正面图在翻转到正面时已经准备好，不会闪一下或卡一下

### 4. 标题和卡片整体下移，彻底避开输入框 — `src/pages/Index.tsx`

不再用现在这个 `240px` 的硬编码顶部值。

改法：

- 给输入框容器加 ref，测量真实高度
- 用“输入框底部 + 64px”作为标题卡片舞台的固定起点
- 标题和卡片作为一个整体一起下移，保持：
  - 输入框到底部标题：64px
  - 标题到卡片：64px

结果：

- 输入框出现后不会再挡住标题
- 标题不会乱跳
- 卡片和标题的相对距离也不会被破坏

### 5. 卡片背面改成你上传的扑克牌背面样式 — `src/components/CardBack.tsx`

这次不再用现在那个近似蓝色图案。

改法：

- 将上传图作为卡片背面的主视觉
- 保留扑克牌白边、圆角、轻阴影，让它更像视频里的卡背
- 正面卡片内容完全不改

如果需要，会只在背面上补非常轻的高光/阴影，不改你提供的主体样式。

### 6. 蒙层上边缘模糊渐变恢复并固定，不再乱改 — `src/components/TemplateCard.tsx`

这次会把蒙层拆成稳定的两层结构，避免又被后续改坏：

- **底层**：底部玻璃/暗色渐变承载层
- **上边缘层**：单独的渐变模糊过渡层

这样做的目的：

- 上边缘模糊不会再因为 3D transform 或层级调整消失
- 卡片正面视觉结构固定，不再随动画方案反复被破坏

我会按现有记忆规则保留：

- 3:4 卡片
- 底部玻璃蒙层
- 上边缘自然渐变模糊
- hover 时文字/按钮切换逻辑不变
